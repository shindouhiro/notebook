1. 核心痛点： 从“阻塞"到"协作”
+ 过去: 递归执行，一旦开始无法中断.长任务直接卡死在主线程,用户交互无响应
+ 现在: 将渲染过程变为同任务处理.它让React拥有了暂停、恢复和丢弃渲染能力

2. 底层架构: 从"树"到"链表"
+ Fiber节点: 即是最小工作单元,也是一种新的静态数据结构
+ 结构: 通过child、sibing和return指针将原来的VDOM树打平成单向链表
+ 意义: 链表结构让React能够通过指针"标记"当前的执行位置,从而实现从中断到恢复

1. 三大关键机制
 + 时间切片: 配置Scheduler，只在浏览器每一帧的空闲时间干活,时间一到就归还主线程
 + 双缓存: 在内存中构建workInProgree树,完成后一次性替换current树.防止渲染中途展示不完整的UI
 + 分阶段执行
    - Render阶段: 深度优先遍历,计算Diff，可中断
    - Commit阶段: 提交DOM更新,同步且不可中断(保证UI原子性)
