
| **维度**             | **SSR (Server Side Rendering)** | **RSC (React Server Components)**   |
| ------------------ | ------------------------------- | ----------------------------------- |
| **主要目标**           | 缩短首屏等待时间 (FCP)，利于 SEO。          | 减少客户端 JS 体积，优化数据获取。                 |
| **运行产物**           | 服务器输出 **HTML 字符串**。             | 服务器输出一种 **特殊的 JSON (RSC Payload)**。 |
| **水合 (Hydration)** | 需要。客户端加载 JS 后接管整个 DOM。          | **不需要**。RSC 在服务器执行完就不再下载到客户端。       |
| **状态保留**           | 页面跳转通常会导致状态丢失（除非是 SPA）。         | 可以在保持客户端状态（如输入框内容）的同时刷新。            |
| **数据交互**           | 必须通过 API (Fetch/REST) 通讯。       | 组件可以直接访问数据库、文件系统。                   |
