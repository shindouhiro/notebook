# Agent Skill 编写完全指南：给初学者的入门篇

欢迎来到 Agent Skill 的世界！如果你想让 AI 助手（如 Cursor、Windsurf 或其他支持 Agentic Coding 的工具）更好地理解你的项目规范、工作流和特定技术栈，编写一份优秀的 **Agent Skill** 是最有效的方式。

本文档将带你从零开始，了解编写一个 Skill 的前置知识、具体步骤以及如何调用它。

---

## 1. 什么是 Agent Skill？

简单来说，**Agent Skill 就是一份给 AI 读的“说明书”**。

它本质上是一个 Markdown 文件（通常命名为 `SKILL.md`）。它包含了特定的系统指令、代码规范、架构要求或解决特定任务的步骤。当 AI 在你的项目中工作时，它会阅读这些 Skill，并按照你设定的偏好和规范来生成代码或回答问题。

**常见的 Skill 场景：**
- **项目级规范**：例如“本项目必须使用 Vue 3 `<script setup>` 和 Tailwind CSS”。
- **特定工作流**：例如“如何构建并推送本项目的 Docker 镜像”。
- **常见问题解答**：例如“处理某个特定报错的固定方案”。

---

## 2. 编写 Skill 的前置知识

编写 Skill 不需要高深的编程技巧，你只需要掌握以下两点：

1. **基本的 Markdown 语法**
   - 知道如何使用标题（`#`, `##`）、列表（`-`）、加粗（`**`）和代码块（` ``` `）来排版内容即可。
   
2. **YAML 前置元数据 (Frontmatter)**
   - 这是放在 Markdown 文件最顶部的配置信息，用一对 `---` 包裹。它主要用于告诉 AI 这个文件的名字和简短描述，帮助 AI 快速建立索引和匹配意图。

3. **清晰的表达能力（Prompt Engineering）**
   - 你需要像指导一个聪明但对你的项目一无所知的新员工一样给 AI 提要求。指令需要**明确、具体、且没有歧义**。

---

## 3. 怎样写一个具体的 Skill？

一个标准的 Skill 通常由两部分组成：**结构目录** 和 **文件内容**。

### 步骤一：创建且了解目录结构
通常，我们需要在一个专门的目录（如 `skills/` 或某个包裹好的 npm 包中）为每个不同的技能创建一个独立文件夹。

一个完整且进阶的 Skill 文件夹通常包含以下结构：

```text
📁 skills/
   📁 vue-rules/
      ├── SKILL.md      # 必需：核心指令与元数据所在文件
      ├── scripts/      # 可选：存放可执行代码（如自动化脚本）
      ├── references/   # 可选：存放相关的官方文档或复杂资料
      └── assets/       # 可选：存放代码模板、UI 资源等辅助文件
```

**它们的作用分别是：**
- `SKILL.md`（**必需**）：Skill 的大脑和入口。这里编写写给 AI 看的系统指令、规范要求和 YAML 配置。
- `scripts/`（**可选**）：当你需要 AI 去运行某些固定脚本时（比如清理缓存、打包发布的特定命令），可以把 `.sh`、`.js` 脚本放在这里，供 AI 在特定任务中调用。
- `references/`（**可选**）：有时候规范很复杂，不适合全部塞在 `SKILL.md` 里，你可以把相关的 API 文档、复杂配置文件作为上下文存储在这里，让 AI 自己去查阅。
- `assets/`（**可选**）：如果你希望 AI 在生成代码时使用特定的图标、或是固定的组件模板结构，就把模板文件放在这。

### 步骤二：编写 YAML Frontmatter
在 `SKILL.md` 文件的最顶端（第 1 行），写入该 Skill 的基本信息：

```yaml
---
name: Vue 3 开发规范
description: 团队内部的 Vue 3 核心开发规范，处理前端工作时必须遵守
---
```
> **注意**：`description` 非常重要！AI 在没有完整读取文件内容前，就是靠这段描述来判断当前任务是否需要加载并阅读这个 Skill 的。

### 步骤三：编写正文内容（给 AI 的指令）
在 `---` 之后，写上具体的规则或流程。内容越有条理，AI 的执行效果越好。

**以下是一个优秀的 `SKILL.md` 示例：**

```markdown
---
name: My Vue Project Rules
description: 团队内部的核心开发规范，处理前端需求时必须严格遵守
---

# Skill: My-Vue-Project-Rules

这是我项目的核心开发规范，当你进行任何前端开发时，请严格遵守以下规则：

## 1. 核心规范
- **语法选择**: 必须使用 `<script setup>` 和 TypeScript。
- **状态管理**: 优先使用 `Pinia`，绝对禁止使用 Vuex。
- **组件命名**: 文件夹和文件名统一使用大驼峰（PascalCase），例如 `UserCard.vue`。

## 2. 样式风格
- **CSS**: 必须使用 Tailwind CSS。请不要在 `<style>` 标签里面写原生 CSS 类，除非迫不得已。

## 3. 代码参考模板
当你需要新建一个组件时，请参考以下结构：
\`\`\`vue
<script setup lang="ts">
import { ref } from 'vue'

const props = defineProps<{
  id: string
}>()

const count = ref(0)
</script>

<template>
  <div class="px-4 py-2 bg-white rounded shadow">
    Hello {{ props.id }}
  </div>
</template>
\`\`\`
结尾提示：请在生成代码后自我检查是否符合上述所有规范。
```
（注：如果你需要更复杂的规则，可以在当前同级目录下提供 `scripts/` 或 `examples/` 文件夹，但初学者完全可以只关注 `SKILL.md` 单文件）。

## 4. 如何调用（使用）一个 Skill？

写好了 Skill，AI 是如何知道并使用它的呢？通常有以下几种方式（取决于你使用的 AI IDE 或 Agent 工具）：

### 方式一：AI 自动匹配与加载（最常见）
在现代的高级 AI 开发工具中，你**不需要手动调用**。
1. 当你给 AI 发送需求：“帮我写一个用户列表界面的组件”。
2. AI 会搜索本地存在的 Skill 列表（查看它们的 `description`）。
3. AI 发现有一个 `name: Vue 3 开发规范` 的技能很符合“组件、前端”这个上下文。
4. AI **自动读取** 这个 `SKILL.md`。
5. AI 根据规则里的要求（使用 TS、TailwindCSS 等）生成符合规范的代码。

### 方式二：显式调用（Slash 命令或 @引用）
如果你想要强制 Agent 执行某个特定的工作流（如发布代码），或者发现 AI 忘记了规范，可以通过显式调用的方法：
- **斜杠命令 (Slash Commands)**：许多进阶的 Agent 工具（包括当前你使用的）支持在对话框中输入 `/` 召唤命令菜单。你可以直接输入 `/skills` 来查看当前挂载的所有可用技能，或者直接输入特定技能名（例如 `/vue-rules`）来**强制唤起并执行**该技能中的特定工作流程和脚本（相当于快捷键）。
- **@提及 (Mention)**：在对话输入框中通过 `@/path/to/SKILL.md` 或 `@技能名`，将这份技能说明书强行挂载到本次对话的上下文中，并告诉它：“按这个规则来做”。

### 方式三：生态继承（Npm Package）
当你把业务做大，可能有多个项目需要共用一套规则。你可以把 `skills/` 目录打包发布（或者使用 workspace），然后通过关联工具将其链接起来，这样换个项目，AI 依然能拥有这些能力。

---

## 5. 初学者的 3 个“避坑”建议

1. **别写太长**：AI 也有上下文长度限制（Token 数量）。尽量提取最核心的规则，不要把几十页的官方文档全复制进去。
2. **明确指示“禁止”事项**：比起告诉 AI “可以做什么”，告诉它“**禁止做什么**”（比如：禁止使用 jQuery、禁止写行内样式）往往效果更好。
3. **提供示例代码**：文字描述有时会产生歧义，而直接给出一个 `Cheat sheet`（代码小抄 / 模板），AI 的模仿能力绝对会让你惊艳。
