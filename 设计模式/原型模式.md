# 原型模式 (Prototype Pattern)

原型模式是一种创建型设计模式，它允许通过**复制（克隆）**现有的对象来创建新对象，而不是通过实例化类。

在 JavaScript 中，原型模式不仅是一种设计模式，更是其**语言的核心基石**（JavaScript 本身就是基于原型的语言）。

---

## 1. 核心概念
- **克隆**：利用一个已有的对象作为“原型”，通过拷贝它的属性和方法来创建一个新实例。
- **性能优化**：如果创建一个新对象的成本很高（例如需要大量的计算或复杂的资源获取），直接克隆现有的对象会更高效。

---

## 2. JavaScript 中的实现方式

### 方式一：Object.create()
这是实现原型模式最标准、最简洁的方式。它创建一个新对象，并将其原型指向现有的对象。

```javascript
const carPrototype = {
  wheels: 4,
  start() {
    console.log("Engine started...");
  },
  stop() {
    console.log("Engine stopped.");
  }
};

// 以 carPrototype 为原型创建一个新对象
const myCar = Object.create(carPrototype);
myCar.color = "Red";

console.log(myCar.wheels); // 4 (继承自原型)
myCar.start(); // Engine started...
```

### 方式二：手动克隆 (Shallow/Deep Copy)
有些场景下，我们需要完全断开与原型的引用关系，生成一个独立的副本。

```javascript
const user = {
  name: "Alice",
  settings: { theme: "dark" }
};

// 1. 浅拷贝 (Shallow Copy)
const shallowClone = { ...user };
// 注意：shallowClone.settings === user.settings 是 true！
// 修改 shallowClone.settings.theme 会直接改变 user.settings.theme

// 2. 深拷贝 (Deep Copy) - 真正的独立副本
const deepCloneNative = structuredClone(user); // 现代环境推荐

// 3. 手写深拷贝实现 (面试与原理深入)
function deepClone(obj, hash = new WeakMap()) {
  if (obj === null || typeof obj !== "object") return obj;
  if (obj instanceof Date) return new Date(obj);
  if (obj instanceof RegExp) return new RegExp(obj);

  // 处理循环引用
  if (hash.has(obj)) return hash.get(obj);

  let cloneObj = new obj.constructor();
  hash.set(obj, cloneObj);

  for (let key in obj) {
    if (Object.prototype.hasOwnProperty.call(obj, key)) {
      cloneObj[key] = deepClone(obj[key], hash);
    }
  }
  return cloneObj;
}
```

> **重要警示**：
> 1. **浅拷贝风险**：在原型模式中，如果原型包含嵌套对象，使用解构赋值等浅拷贝手段会导致所有克隆体**共享同一个深层引用**。
> 2. **原生支持**：现代开发优先使用 `structuredClone()`，它是浏览器内置的，采用了高效的结构化克隆算法。
> 3. **手写要点**：在面试中手写深拷贝，必须考虑到 **循环引用**（使用 `WeakMap`）和 **特殊对象**（如 Date, RegExp）的处理。

---

## 3. 使用场景

### 1. 性能优化：共享方法
在 JS 中，如果你有 1000 个实例，普通的类实现如果把方法写在 constructor 里，每个实例都会拥有一份方法的拷贝；而使用原型（Prototype），所有实例共享同一套方法，极大地节省了内存。

### 2. 初始化成本高昂的对象
当一个对象的创建过程包含复杂的数据库查询、大量的数学计算或网络同步时，通过克隆一个已经初始化好的“模板对象”会快得多。

### 3. 多种变体对象的创建
假设你在开发一个游戏，需要创建很多种“怪兽”。它们大部分属性相同，只有颜色和血量不同。你可以先创建一个 `Monster` 原型，然后通过克隆并微调参数来快速生成各种变体。

### 4. 插件与配置继承
在很多框架设计中，默认配置（Default Config）就是原型。用户的自定义配置通过继承默认配置来实现：
```javascript
const defaultConfig = { timeout: 3000, retry: true };
const userConfig = Object.create(defaultConfig);
userConfig.timeout = 5000; // 覆盖默认，但 retry 依然是 true
```

---

## 4. 深度思考：原型模式与类的区别

| 特性 | 类 (Constructor/Class) | 原型模式 (Object.create) |
| :--- | :--- | :--- |
| **关注点** | 关注“对象是什么”（结构） | 关注“对象像什么”（实例） |
| **灵活性** | 结构相对固定 | 可以动态地给原型添加或修改属性 |
| **性能** | 创建新实例涉及构造逻辑执行 | 直接拷贝内存空间或通过引用链查找，通常更快 |

---

## 5. 优缺点分析

### 优点：
- **性能卓越**：减少了实例化对象时的构造函数执行开销。
- **动态性**：可以在运行时动态改变原型，从而影响所有子实例。
- **简化创建过程**：不需要复杂的类继承体系。

### 缺点：
- **深拷贝问题**：如果原型对象包含引用类型的属性（如数组、嵌套对象），简单的克隆只能实现浅拷贝，修改子对象可能会意外地影响到原型或其他子对象。
- **逻辑隐蔽**：原型继承链如果过长，调试时可能比较难追踪属性到底定义在哪里。

---

> **面试金句**：JavaScript 的核心就是原型模式。类（Class）语法糖背后依然是 `prototype` 链接。理解原型模式就是理解 JS 的内存分配与对象继承机制。
